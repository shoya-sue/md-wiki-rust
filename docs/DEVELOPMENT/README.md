## 開発ガイドライン

### コミット規約

コミットメッセージは日本語で記述し、以下の形式に従います：

```
機能名/修正内容の要約

詳細な説明（必要な場合）
```

例：
- `検索機能を追加`
- `バックエンドのエラーハンドリングを改善`

### ブランチ戦略

- `main`: 安定版のコード
- `develop`: 開発中のコード
- `feature/機能名`: 新機能の開発
- `fix/バグ名`: バグ修正

### コーディング規約

- Rustのコードは`rustfmt`でフォーマット
- JavaScriptのコードはESLintと標準のフォーマッターを使用
- 関数には適切なコメントを追加
- 複雑なロジックには説明コメントを追加

### テスト

- 各機能にはユニットテストを追加
- APIエンドポイントには統合テストを追加
- GUIコンポーネントにはスナップショットテストを検討
